# NDP120 资料获取与功耗验证计划

## 已取得

- `NDP120_Product_Brief.pdf`
- 公开开发平台信息：Avnet RASynBoard、Arduino Nicla Voice。

Product Brief 可确认 NDP120 面向 always-on audio/sensor fusion，具有多路 PDM、I2S/TDM、DSP/MCU 等资源，但不足以完成量产原理图和功耗保证。

## 向 Syntiant / 授权代理索取的完整资料

必须逐项索取并记录版本号：

1. NDP120 Full Datasheet / Product Specification。
2. Hardware Integration Guide。
3. Package drawing、land pattern、pin map、最大高度。
4. Recommended Operating Conditions、Absolute Maximum Ratings。
5. 每个电源轨的电压、峰值电流、上电/掉电时序和去耦要求。
6. PDM/I2S/TDM/SPI/UART/GPIO 电平与时序。
7. Boot ROM、外部 Flash、固件下载和恢复流程。
8. Reference schematic、BOM、layout guide、IBIS（若提供）。
9. Syntiant TDK/SDK、编译工具、模型转换工具和许可证。
10. Wake-word 模型的语言、授权、训练和更新方式。
11. 各运行模式的电流：deep sleep、监听、推理、host transfer、固件更新。
12. QFN 与 WLBGA 的可采购料号、MOQ、交期与生命周期。

## 推荐开发套件

### 主选：Avnet RASynBoard

原因：板上包含 NDP120、数字麦克风、IMU、SPI Flash 和主控，适合验证声学事件、固件加载和 host wake 流程。

### 辅助：Arduino Nicla Voice

原因：容易采购，可快速验证 NDP120 关键词识别和 BLE 触发；但其主控与电源架构不代表最终 nRF54L15 AON 板。

## 功耗测试连接

```text
可编程电源 / Otii / Joulescope
        |
   独立测量 NDP120 rail
        |
NDP120 + 1颗 Wake Mic
        |
WAKE GPIO -> nRF54L15 DK
```

不要只测整块 EVK USB 输入功耗，否则 USB-UART、LED、主 MCU 和稳压器会污染结果。

## 必测状态

| 状态 | 条件 | 记录 |
|---|---|---|
| Power-off leakage | NDP rail 关闭，所有 IO 保持真实系统状态 | 漏电、是否反向供电 |
| Deep sleep | 不监听，保留可唤醒资源 | 平均/峰值电流 |
| Continuous listening | 1颗 PDM Mic，真实采样率 | 10分钟平均、时域波形 |
| Inference | 每分钟 1/10/60 次触发 | 单次能量、延迟 |
| Host transfer | SPI/UART 发送结果 | 峰值、持续时间 |
| False wake test | 安静/风噪/音乐/骑行噪声 | 每小时误唤醒次数 |
| Temperature | 0/25/45°C | 功耗、识别率 |
